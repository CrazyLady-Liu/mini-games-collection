# 音序回响 - 音乐记忆游戏规范

## 1. 项目概述

- **项目名称**: 音序回响 (Sequence Echo)
- **项目类型**: 音乐记忆类HTML5游戏
- **核心功能**: 玩家聆听系统播放的音符序列，然后按顺序复刻
- **目标用户**: 休闲游戏玩家，音乐爱好者

## 2. 视觉与渲染规范

### 场景设置
- **背景**: 深色渐变背景 (#1a1a2e → #16213e)，营造沉浸感
- **布局**: 居中布局，垂直排列标题、音符按钮区、状态信息

### 音符按钮设计
- **数量**: 固定4个音符按钮
- **尺寸**: 100px × 100px 圆形按钮
- **样式**:
  - 极简扁平风格
  - 圆角: 50% (完全圆形)
  - 低饱和马卡龙配色:
    - 音符1: 薄荷绿 #a8e6cf
    - 音符2: 樱花粉 #ffb7b2
    - 音符3: 淡紫色 #c9b1ff
    - 音符4: 柠檬黄 #ffeaa7
  - 阴影: 柔和投影，增加立体感
  - 默认状态: 正常大小
  - 高亮状态: 放大1.1倍 + 发光效果 + 震动动画

### 动画效果
- **高亮震动**: 当音符播放时，按钮缩放1.1倍 + CSS shake动画
- **过渡动画**: 所有状态变化使用0.3s ease过渡
- **成功反馈**: 绿色闪烁
- **失败反馈**: 红色闪烁

### UI元素
- **标题**: 游戏名称，居中显示
- **关卡显示**: 当前关卡数字
- **得分显示**: 连续正确次数
- **状态提示**: 当前游戏状态（准备/播放/输入/结果）

## 3. 游戏逻辑规范

### 关卡难度递增
- **第1-2关**: 3个音符序列
- **第3-4关**: 4个音符序列
- **第5-6关**: 5个音符序列
- **第7关起**: 6个音符序列
- **音符范围**: 1-4 (对应4个按钮)

### 播放演示模块
- **播放模式**: 系统自动循环播放序列
- **播放间隔**: 每个音符800ms，音符间停顿400ms
- **高亮效果**: 播放时对应按钮高亮+震动
- **播放速度**: 缓慢，便于玩家记忆

### 玩家交互模块
- **输入模式**: 玩家按听到的顺序点击按钮
- **音效反馈**: 每次点击播放对应音符音调
- **实时校验**: 实时检查点击顺序是否正确
- **容错机制**:
  - 错序时显示错误，允许重试当前关卡
  - 不立即判定失败，可重新开始本关
- **输入完成**: 全部正确后进入下一关

## 4. 音频规范

### 音符音调
- 使用Web Audio API合成音符
- 音符1: C4 (261.63 Hz)
- 音符2: E4 (329.63 Hz)
- 音符3: G4 (392.00 Hz)
- 音符4: B4 (493.88 Hz)
- **音色**: 柔和的正弦波，带轻微衰减

### 音效
- **点击音效**: 短促的音符音调
- **成功音效**: 上升音阶
- **失败音效**: 下降音阶

## 5. 交互规范

### 游戏流程
1. **开始界面**: 显示"开始游戏"按钮
2. **播放阶段**: 显示"请聆听..."，自动播放序列
3. **输入阶段**: 显示"请复刻序列"，玩家点击按钮
4. **结果反馈**: 显示成功/失败信息
5. **下一关/重试**: 自动或手动进入

### 用户操作
- **点击音符按钮**: 触发对应音符
- **开始游戏按钮**: 初始化并开始第一关
- **重新开始按钮**: 失败后可重新尝试
- **重听序列按钮**: 在输入阶段可重新播放当前序列
- **重试本关按钮**: 重新开始当前关卡

## 6. 技术实现

### 技术栈
- 纯HTML5 + CSS3 + JavaScript
- Web Audio API (音频合成)
- 无外部依赖库

### 文件结构
- 单文件 `index.html` (包含所有代码)
